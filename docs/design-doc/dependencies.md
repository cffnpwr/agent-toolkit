# 外部ランタイム依存

本リポジトリのスキル・hookが必要とする外部ランタイム依存（依存パッケージ・汎用CLI・インタプリタなど）を、どう宣言・供給・検証するかの方針を定める。
基本となる方針（ホスト充足前提、明示宣言と再現性、再現的な操作のみ自律実行）は[設計原則](./principles.md)に定める。本節はその宣言の具体を扱う。

スキルとhookは依存の分類（依存パッケージ・外部ツール）と供給方式（バンドルしたロックファイルからの同期・外部ツールの存在確認）を共有する。差は宣言の場所と、同期を起動する主体・依存を満たせないときの扱いに限られる。

## 宣言の場所

| 成果物 | 宣言場所 |
| --- | --- |
| スキル | `SKILL.md`の`compatibility`フロントマターフィールドと本文の`## Requirements`節（内容を一致させる） |
| hook | 当該hookのREADME（`.apm/hooks/scripts/<hook名>/README.md`）の`## Requirements`節 |

hookは`SKILL.md`を持たず、マニフェストが`.json`のため（[全体アーキテクチャ](./architecture.md)）、スキルの二層宣言（`compatibility`＋`## Requirements`）は適用しない。

## 宣言が必要なとき

外部依存（依存パッケージ、または外部ツール）があるときに宣言し、外部依存が無い成果物では省略する。
外部ツールへの依存はファイルに現れないため、ファイルの有無だけでは判定しない。
同梱した定義ファイル・ロックファイル・`scripts/`配下の言語ファイルに加え、成果物のロジックが呼び出す外部CLI・インタプリタ（`jj`・`gh`・`git`等）も依存として数える。

## 依存の分類

外部依存を「依存パッケージ」と「外部ツール」の2種に分類し、それぞれ別の規約で扱う。

### 依存パッケージ

プログラミング言語のライブラリ・パッケージとして、その言語のパッケージマネージャで導入するもの。
成果物に同梱した定義ファイルとロックファイルでバージョンを固定し、使用前にロックファイルから同期する。
具体例は textlintとそのプラグイン群、markdownlint-cli2、PyYAMLなど。

言語ごとのパッケージマネージャを次に統一する。

| 言語 | パッケージマネージャ | 依存定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| JavaScript / TypeScript | bun | `package.json` | `bun.lock` | `bun install --frozen-lockfile` |
| Python | uv | `pyproject.toml` | `uv.lock` | `uv sync --frozen` |

この表は、成果物が実行時に使う言語ライブラリを同梱して固定する場合に適用する（スキル・hook共通）。
配布物の供給は実行時依存のみを対象とし、開発依存（lint・型チェック・テスト等）は含めない（`--production`等で除外する）。
「配布時にインストール成功を保証」とは、配布された状態であれば記載のインストールコマンドを実行すれば必ず成功することを指す。自動インストールや成果物の同梱ではない。
新しい言語を同梱する場合は、その言語のパッケージマネージャを1つ選んで本表に追記する。

### 外部ツール

依存パッケージ以外の外部依存。
言語のパッケージマネージャで導入するライブラリではなく、汎用CLI・インタプリタ・パッケージマネージャ本体などのバイナリ。
具体例は gh、jq、jj、node、bun、uvなど。

- バージョン要件とともに全列挙する。
- 使用前の存在確認コマンドを明記する（スキルのみ。hookは起動スクリプトが存在確認を実装するため、宣言への明記は不要）。
- インストール方法は記載しない（入手方法は環境ごとに異なるため、成果物の責務としない）。
- 不在時の扱いは成果物で異なる（後述）。

## 依存の供給

スキルとhookは同じ供給方式を採る。依存パッケージはバンドルしたロックファイルから使用前に同期し、外部ツールは存在確認する。
差は、同期を起動する主体と、依存を満たせないときの扱いの2点のみで、成果物が対話的に使われるか非対話で発火するかという1つの軸から派生する。

| 成果物 | 同期を起動する主体 | 依存不足・同期失敗時 |
| --- | --- | --- |
| スキル | 使用時にAI Agent（対話） | 進行を停止しユーザーへエスカレーションする |
| hook | 発火時に起動スクリプト（非対話） | fail-openで警告して通す（導入はユーザーに委ねる） |

### 依存パッケージの同期

成果物に定義ファイルとロックファイルを同梱し、使用前にロックファイルから同期する。
`.venv`・`node_modules`などgit管理外のみを変更しgit検知の更新を生まない再現的操作のため、停止せず実行してよい（[設計原則](./principles.md)）。

```sh
# ロックファイルから実行時依存を同期する。
uv sync --frozen --no-dev                   # Python
bun install --frozen-lockfile --production  # JavaScript / TypeScript
```

hookは発火ごとにfrozen同期する。`bun install`は同期済みなら確認のみで安価に済むため毎回実行してよく、これによりファイル存在に基づく判定の偽陰性を避ける。
開発ツーリング（lint・型チェック・テスト）の依存は配布物の実行時依存ではなく`--production`同期から除外される。これらはリポジトリでの開発時にのみ全依存を同期して使い、配布物の供給とは区別する。

### 外部ツールの存在確認

使用前に存在確認のみ行う。導入・代替手段へのフォールバックは行わない。

```sh
command -v gh >/dev/null 2>&1 || { echo "gh not found." >&2; exit 1; }
```

不在時の扱いは上表のとおり成果物で異なる（スキルは停止、hookはfail-open）。
スキルの宣言テンプレートは[スキル機構](./skills.md)に示す。
