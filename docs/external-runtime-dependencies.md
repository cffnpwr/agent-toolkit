# 外部ランタイム依存の供給方式

本リポジトリのAgent Skillsが必要とする外部ランタイム依存（依存パッケージ・汎用CLI・インタプリタなど）を、どう供給・宣言・検証するかの方針を定める。
APMはシステムバイナリや依存パッケージを供給しない。
そのため各スキルが自身の依存を明示し、ホスト側で充足されることを前提に動作する。

この方針は現行スキルだけでなく、本リポジトリに今後追加されるすべてのスキルに適用する一般則である。

## 基本原則

- ユーザーからの明示的な指示がある場合を除き、AI Agentによる依存の導入・修正・フォールバックの禁止。依存の充足は配布・環境側の責務。
- 使用前の依存充足の確認と、不足時の進行停止・ユーザーへのエスカレーション。Agentによる無断インストールや代替手段への切り替えの禁止。
- 全依存の`## Requirements`節での明示的な宣言。

## 依存の分類

外部依存を「依存パッケージ」と「外部ツール」の2種に分類し、それぞれ別の規約で扱う。

### 依存パッケージ

プログラミング言語のライブラリ・パッケージとして、その言語のパッケージマネージャで導入するもの。
具体例は textlintとそのプラグイン群、markdownlint-cli2、PyYAMLなど。

以下の規約に従う。

- 依存定義ファイルとロックファイルによる管理。両者をスキルに同梱する。
- パッケージマネージャの言語ごとの統一（下表参照）。
- スキルへのインストールコマンドの明記。
- 定義ファイルとロックファイルによるバージョン固定と、依存解決の再現性確保。
- 「配布時にインストール成功を保証」とは、配布された状態であれば記載のインストールコマンドを実行すれば必ず成功することを指す。自動インストールや成果物の同梱ではない。
- 使用前のインストール確認。未インストールがあれば停止・エスカレーション（Agentは導入しない）。

言語ごとのパッケージマネージャを次に示す。

| 言語 | パッケージマネージャ | 依存定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| JavaScript / TypeScript | pnpm | `package.json` | `pnpm-lock.yaml` | `pnpm install` |
| Python | uv | `pyproject.toml` | `uv.lock` | `uv sync` |

新しい言語のスキルを追加する場合は、その言語のパッケージマネージャを1つ選んで本表に追記する。
そのうえで依存パッケージの規約に従う。

### 外部ツール

依存パッケージ以外の外部依存。
言語のパッケージマネージャで導入するライブラリではなく、汎用CLI・インタプリタ・パッケージマネージャ本体などのバイナリ。
具体例は gh、jq、node、pnpm、uvなど。

以下の規約に従う。

- バージョン要件とともにスキル内へ全列挙。
- 使用前の存在確認コマンドの明記。
- 不在時の停止・エスカレーション（Agentは導入しない）。
- インストール方法のスキルへの非記載。入手方法は環境ごとに異なるため、スキルの責務としない。

## 具体例

### 依存パッケージ（pnpm）

スキルに`package.json`と`pnpm-lock.yaml`を同梱する。

インストールコマンドは次のとおり（使用前にユーザー側で実行されている前提）。

```sh
pnpm install
```

使用前の確認は次のとおり（未インストールなら停止・エスカレーション）。

```sh
# 依存が解決済みか確認する。未解決なら pnpm installをユーザーに促して停止。
pnpm ls --depth -1 >/dev/null 2>&1 || {
  echo "依存が未インストール。'pnpm install'を実行してから再試行すること。" >&2
  exit 1
}
```

### 依存パッケージ（uv）

スキルに`pyproject.toml`と`uv.lock`を同梱する。

インストールコマンドは次のとおり。

```sh
uv sync
```

使用前の確認は次のとおり。

```sh
# 同期済みか確認する。未同期なら uv syncをユーザーに促して停止。
uv sync --check >/dev/null 2>&1 || {
  echo "依存が未同期。'uv sync'を実行してから再試行すること。" >&2
  exit 1
}
```

### 外部ツール

スキルに導入方法は書かず、存在確認のみ行う。

```sh
# 存在確認とバージョン要件のチェック。欠けていれば停止・エスカレーション。
command -v gh >/dev/null 2>&1 || { echo "ghが見つからない。" >&2; exit 1; }
command -v jq >/dev/null 2>&1 || { echo "jqが見つからない。" >&2; exit 1; }
```

## `## Requirements`テンプレート

各スキルの`SKILL.md`に以下の`## Requirements`節を置き、依存パッケージと外部ツールの依存を宣言する。
該当しない区分は省略してよい。

````markdown
## Requirements

使用前に以下の依存を確認すること。
ひとつでも欠けていれば進行を停止し、ユーザーへエスカレーションする。
Agentは導入・修正・フォールバックを行わない。

### 依存パッケージ

| 言語 | パッケージマネージャ | 定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| <言語> | <pm> | <定義ファイル> | <ロックファイル> | `<install cmd>` |

確認コマンドを次に示す。

```sh
<インストール済みか確認するコマンド>
```

### 外部ツール

| ツール | バージョン要件 |
| --- | --- |
| <tool> | `>= x.y` |

存在確認コマンドを次に示す。

```sh
<command -v などによる存在確認>
```
````
