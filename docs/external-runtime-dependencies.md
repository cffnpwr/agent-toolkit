# 外部ランタイム依存の供給方式

本リポジトリのAgent Skillsが必要とする外部ランタイム依存（依存パッケージ・汎用CLI・インタプリタなど）を、どう供給・宣言・検証するかの方針を定める。
APMはシステムバイナリや依存パッケージを供給しない。
そのため各スキルが自身の依存を明示し、ホスト側で充足されることを前提に動作する。

この方針は現行スキルだけでなく、本リポジトリに今後追加されるすべてのスキルに適用する一般則である。

## 基本原則

- 一般則: リポジトリ内でgitが検知するファイル更新を生まずに実行できる操作は、AI Agentが自律実行してよい。コミット済みロックファイルからの依存同期（`uv sync --frozen`・`pnpm install --frozen-lockfile`等）は`.venv`・`node_modules`等のgit管理外のみを変更するため、これに該当し許可する。
- 依存の追加・更新（定義ファイル・ロックファイルの変更）、外部ツールの導入、代替手段へのフォールバックの禁止。これらが必要な場合は進行を停止しユーザーへエスカレーションする。依存宣言の変更と外部ツールの充足は配布・環境側の責務。
- 全依存の二重宣言。`SKILL.md`フロントマターの`compatibility`フィールド（機械可読な要約）と本文の`## Requirements`節（正本となる詳細）の両方に記し、両者を一致させる。

## 依存宣言の二層

依存宣言は次の2つで構成し、内容を一致させる。

- `compatibility`フロントマターフィールド: 依存の1〜2行の機械可読な要約。
- `## Requirements`節: 依存パッケージ表（定義ファイル・ロックファイル・インストールコマンド）と外部ツール表（バージョン要件・存在確認）からなる正本。

両者とも、依存（同梱スクリプト・依存パッケージ・外部CLI）がある場合に必須とし、依存が無いスキルでは省略する。
依存の有無は確実に観測可能なファイル存在（依存定義ファイル・ロックファイル・`scripts/`配下の言語ファイル）で判定する。

## 依存の分類

外部依存を「依存パッケージ」と「外部ツール」の2種に分類し、それぞれ別の規約で扱う。

### 依存パッケージ

プログラミング言語のライブラリ・パッケージとして、その言語のパッケージマネージャで導入するもの。
具体例は textlintとそのプラグイン群、markdownlint-cli2、PyYAMLなど。

以下の規約に従う。

- 依存定義ファイルとロックファイルによる管理。両者をスキルに同梱する。
- パッケージマネージャの言語ごとの統一（下表参照）。
- スキルへのインストールコマンドの明記。
- 定義ファイルとロックファイルによるバージョン固定と、依存解決の再現性確保。
- 「配布時にインストール成功を保証」とは、配布された状態であれば記載のインストールコマンドを実行すれば必ず成功することを指す。自動インストールや成果物の同梱ではない。
- 使用前にロックファイルから依存を同期する（`uv sync --frozen`等）。同期はgit管理外のみを変更するためAgentが実行してよい。依存の追加・更新が必要な場合は停止・エスカレーション。

言語ごとのパッケージマネージャを次に示す。

| 言語 | パッケージマネージャ | 依存定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| JavaScript / TypeScript | pnpm | `package.json` | `pnpm-lock.yaml` | `pnpm install --frozen-lockfile` |
| Python | uv | `pyproject.toml` | `uv.lock` | `uv sync --frozen` |

新しい言語のスキルを追加する場合は、その言語のパッケージマネージャを1つ選んで本表に追記する。
そのうえで依存パッケージの規約に従う。

### 外部ツール

依存パッケージ以外の外部依存。
言語のパッケージマネージャで導入するライブラリではなく、汎用CLI・インタプリタ・パッケージマネージャ本体などのバイナリ。
具体例は gh、jq、node、pnpm、uvなど。

以下の規約に従う。

- バージョン要件とともにスキル内へ全列挙。
- 使用前の存在確認コマンドの明記。
- 不在時の停止・エスカレーション（Agentは導入しない）。
- インストール方法のスキルへの非記載。入手方法は環境ごとに異なるため、スキルの責務としない。

## 具体例

### 依存パッケージ（pnpm）

スキルに`package.json`と`pnpm-lock.yaml`を同梱する。

インストールコマンドは次のとおり。

```sh
pnpm install --frozen-lockfile
```

使用前に依存をロックファイルから同期する。`node_modules`のみを変更しgit検知の更新を生まないため、Agentが実行してよい。

```sh
# ロックファイルから依存を同期する（Agentが実行してよい）。
pnpm install --frozen-lockfile
```

### 依存パッケージ（uv）

スキルに`pyproject.toml`と`uv.lock`を同梱する。

インストールコマンドは次のとおり。

```sh
uv sync --frozen
```

使用前に依存をロックファイルから同期する。`.venv`のみを変更しgit検知の更新を生まないため、Agentが実行してよい。

```sh
# ロックファイルから依存を同期する（Agentが実行してよい）。
uv sync --frozen
```

### 外部ツール

スキルに導入方法は書かず、存在確認のみ行う。

```sh
# 存在確認とバージョン要件のチェック。欠けていれば停止・エスカレーション。
command -v gh >/dev/null 2>&1 || { echo "gh not found." >&2; exit 1; }
command -v jq >/dev/null 2>&1 || { echo "jq not found." >&2; exit 1; }
```

## 依存宣言テンプレート

各スキルの`SKILL.md`に、フロントマターの`compatibility`フィールドと本文の`## Requirements`節を置き、依存パッケージと外部ツールの依存を宣言する。
該当しない区分は省略してよく、依存が無いスキルでは両方を省略する。

フロントマターの`compatibility`フィールドは次のとおり（1〜2行の要約）。

````markdown
compatibility: |
  Required: <インタプリタ・外部ツール>; packages: <パッケージ名>（<定義ファイル> / <ロックファイル> 経由）
  Scripts: <同梱スクリプト名>
````

本文の`## Requirements`節は次のとおり。

````markdown
## Requirements

使用前に依存パッケージをロックファイルから同期する（`uv sync --frozen`等。git管理外のみを変更するためAgentが実行してよい）。
依存の追加・更新、外部ツールの導入、代替手段へのフォールバックが必要な場合は、進行を停止しユーザーへエスカレーションする。

### 依存パッケージ

| 言語 | パッケージマネージャ | 定義ファイル | ロックファイル | インストールコマンド |
| --- | --- | --- | --- | --- |
| <言語> | <pm> | <定義ファイル> | <ロックファイル> | `<install cmd>` |

同期コマンドを次に示す。

```sh
<ロックファイルから同期するコマンド>
```

### 外部ツール

| ツール | バージョン要件 |
| --- | --- |
| <tool> | `>= x.y` |

存在確認コマンドを次に示す。

```sh
<command -v などによる存在確認>
```
````
