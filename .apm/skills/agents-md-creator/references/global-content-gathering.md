# 内容収集（グローバルモード・面談駆動）

グローバル設定は大半が**コードから読めないユーザーの選好（意図）**で構成される。解析対象のリポジトリが存在しないため、リポジトリモードのハイブリッド方式（コードから事実を読む）は成立しない。**面談で収集する**のが基本。

## ステップ1: 既存設定と環境の読み取り（補助）

面談の前に、取り込める材料があれば読む。あくまで下書きの種であり、これだけでは埋めない。

### 既存グローバル設定

検出した各ツールのグローバル設定を読み、内容を取り込む。

- ツール中立な実体ファイル: `~/.config/agents/AGENTS.md`（既定の配置先）。
- 各ツールのネイティブ設定: `~/.claude/CLAUDE.md`、`~/.codex/AGENTS.md` 等。
- これらが symlink の場合はリンク先（dotfilesリポジトリ等）の実体を読む。
- 既存設定が `@`参照・プレーン参照で分割している `docs/*.md` も併せて読む。

検出と配置の詳細は [global-entrypoint.md](global-entrypoint.md) を参照。

### 環境事実（ツール使用方針の補助のみ）

- OS・シェル・パッケージマネージャ（例: nix管理か、Homebrewか）。
- これらは「ツール使用方針」セクションの下書きの補助に使う。方針そのものはユーザー確認。

## ステップ2: 面談（中心）

[global-recommended-skeleton.md](global-recommended-skeleton.md) のセクションに沿って、ユーザーの選好を確認する。既存設定から取り込めた分は下書きとして示し、不足分を質問で埋める。

確認する主な観点:

- **コミュニケーション・言語方針**: 応答言語、文体、トーン。
- **自律性・権限**: 確認なしで実行してよい範囲、破壊的操作の扱い、VCSの自律性。
- **ツール使用方針**: 優先ツール、インストール方針、調査手段の優先順位。
- **VCSデフォルト**: 既定VCS、ブランチ運用。
- **応答スタイル**: 出力フォーマット、報告の粒度、禁止する言い回し。
- **横断的禁止事項・制約**: 全プロジェクトで守る制約。

## 原則

- グローバル設定は選好の集合。**推測でデフォルト値や方針を捏造しない**。
- 既存設定から読めた事実はユーザーに聞かない。読めない選好は推測で埋めず確認する。
- 既存設定 → 不足分の確認、の順で進め、最初の一括質問攻めを避ける。
- 特定プロジェクトに紐づく事実（ビルドコマンド・ディレクトリ構成等）はグローバルには書かない。
